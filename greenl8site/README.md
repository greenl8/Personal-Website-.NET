## Local Development

### Prerequisites
- .NET 9.0 SDK
- Node.js 18+ and npm
- Git

### Running Locally
1. Clone the repository
2. Navigate to the project directory
3. Restore .NET dependencies: `dotnet restore`
4. Install npm packages: `cd greenl8web-client && npm install`
5. Build Angular app: `npm run build`
6. Return to root: `cd ..`
7. Run the application: `dotnet run`

## Deployment

This application is configured for deployment on **Render.com** with **Railway PostgreSQL**.

Important: No secrets are committed to this repository. Configure all sensitive values via environment variables in your hosting provider.

### Quick Setup
Run the setup script to generate secure credentials:

**Windows (PowerShell):**
```powershell
.\setup-deployment.ps1
```

**Linux/macOS (Bash):**
```bash
chmod +x setup-deployment.sh
./setup-deployment.sh
```

### Deployment Steps

1. **Set up Railway PostgreSQL database**
   - Create account at [Railway](https://railway.app)
   - Provision PostgreSQL service
   - Copy the connection URL

2. **Deploy to Render.com**
   - Create account at [Render.com](https://render.com)
   - Connect your Git repository
   - Configure as Docker web service
   - Set environment variables (generated by setup script)

3. **Complete deployment guide**
   - See [DEPLOYMENT.md](DEPLOYMENT.md) for detailed instructions

## Environment Variables

### Required for Production
- `DATABASE_URL` - PostgreSQL connection string (e.g., `postgresql://USER:PASSWORD@HOST:PORT/DBNAME`)
- `TOKEN_KEY` - JWT signing key (32+ characters)
- `ADMIN_USERNAME` - Admin user username
- `ADMIN_EMAIL` - Admin user email
- `ADMIN_PASSWORD` - Admin user password
- `BASE_URL` - Application base URL (e.g., `https://your-service-name.onrender.com`)

### Development
Uses SQLite database with default settings in `appsettings.Development.json`.

Copy `appsettings.example.json` to `appsettings.json` (or set env vars) and fill in values before running in production.

## Admin Access

After deployment, an admin user is automatically created using the environment variables you set. Use these credentials to log in and start creating content.
